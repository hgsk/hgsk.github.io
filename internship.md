# 見習い

今から800年ほど昔、中世ヨーロッパでは、若者が「見習い」として職場に参加し、労働しながら「親方」の元で仲間と共同生活を続けました。

数年間、この暮らしを続け、技術や知識を身につけることでギルドに認められ、その道で働くことを許されました。

「見習い」は、現代でも盛んに行われています。

若者が、職場体験を通じて技術を身につけることで、そのまま就職に繋がっていく仕組みです。

工業、医療、エンジニアリングなど、「見習い」が盛んに行われている分野には特徴があります。この分野の作業には、危険が伴う、ミスが社会的に許されない、といった性質があり、責任能力が強く求められます。

具体的には、もし失敗してしまった場合は、全力で改善と、再発防止に努めることが求められます。

そのためこういった環境では、人的エラーを極力少なくするために、単に高度な技術を身につけるだけでなく、現場で他の技術者と信頼関係を結び、困難な状況でも団結して立ち向かうことのできる、コミュニケーション能力も求められます。

その点、「見習い」には、共同生活の中で「仲間」を作り信頼関係を築くという側面があります。

気をつけなければならないのは、「見習い」は多かれ少なかれ若者の自由を縛る、ということです。

中世のギルドは支配的で、当時は若者にとって過酷な環境でした。

今も過酷な職場はあるでしょう。

とはいえ、労働は民主化され、法律で守られるようになり、状況はだいぶ良くなりました。

それでも「見習い」は、ともすれば危険な失敗をしかねません。「親方」から制限を受けたり、叱られたり、守られながら仕事をします。

いずれにせよ、習うより慣れろ、ということわざもありますし、とにかく試しにやってみる、という入り口があることは、若者にとっては、社会経験を積む重要な選択肢の一つと言えます。

エンジニア業界は、ずっと人材不足です。

もしあなたが、コンピュータにちょっとでも興味があれば、どこかのIT企業に、アルバイトや派遣社員として、すぐに入れるでしょう。

私もそんな風にこの業界に入りました。

「未経験者OK」の会社がたくさんあります。

多くの人に入り口が開かれている、という意味では、この業界にも「見習い」に似た仕組みがあるのかもしれません。

ただし、人材不足は指導者不足でもあります。

職場に、あなたの面倒をちゃんと見てくれる人はいないかもしれません。

もし仕事が合わなかったり、心を痛めるほど辛い思いをするなら、さっさと辞めて、また次の職場を探しましょう。

ちなみに、インフィニットループはいつでもアルバイトを募集しています！

弊社には人材育成担当がおり、仕事をしていて困ったら気軽に相談できる環境です。

担当者は首都圏でエンジニアやマネージャーとして一定のキャリアを積んでいます。

興味のある方はぜひご連絡ください！

# インターンシップ

さて、19世紀に入り、職業人を育てる新しい取り組みが、医療の分野で始まりました。

インターンシップです。

この取り組みは、医療という、社会的責任の大きい分野の大学の医療実習制度の一貫として行われました。

この制度が確立し、病院で実際に働きながら単位を修得して医者になる、という「研修医」の仕組みができました。

その後、一般企業がこの仕組みに注目しました。

1960年代から現在のような企業インターンシップが始まりました。

学生は、自分の専門分野に近い高度な技術を活用している企業や団体に関わって、共同作業を行うようになりました。

このように、「インターンシップ」は、「見習い」とは異なる歴史をもつ、近代的な制度です。

若者が最初のキャリアを積む、という意味では同じようなものですから、混同されることも多いかもしれません。

現在、アメリカにはキャリアを積むためにインターンシップに参加する学生が30万人以上いるそうです。

大学や専門学校と企業が連携して、学生のカリキュラムの一環として、インターンシップを提供しています。

こうなると、一方でインターンシップに参加していない学生、大学や専門学校に通えない貧しい学生は、圧倒的に不利になってしまいます。

こんな社会問題が現れるほどに、社会に浸透しているのが、アメリカのインターンシップ制度です。

日本では、まだインターンシップは社会全体の仕組みとして確立されていません。

教育の量や質の問題は取りざたされますが、引きこもりに代表されるような若者自身のキャリア形成の失敗は、自己責任論や家族問題と結びついて個別に取り上げられる傾向があり、大きな社会問題になりません。

そんな認識のもとで、私なりに、学生に受けてほしいインターンシップとは何かを検討し、今回のカリキュラムを考えました。

# チームの紹介

今年のインターンシップでは、15人チームを作り、2週間、VRゲームを作ってもらいました。

15人というと、ちょっとしたベンチャー企業や、大企業の1部署に相当する規模です。

参加者の皆さんは、すでに、それぞれ学校で研究をしたりしていて、一定の技術を備えていました。

そのため今回は、大きいチームを作ることで、コミュニケーションやスケジューリングなど、現実的な形で経験してもらうことを狙いました。

安全な形で業務に触れてもらうインターンシップにもすることもできます。

ですが今回は、エンジニアリングの本質的な部分として「技術やアイデアを、どうやって他人と共有して成果につなげるか」という点を実際に肌で感じてもらうことが、エンジニアを目指す学生にとって、新鮮な体験になると考えました。

これまで私が出会った中では、エンジニアとして苦労する人は、真面目だけど会話が苦手なタイプに多く、伸びる人は、ズボラだけど社交的なタイプが多いです。

エンジニアとしての適性は、コミュニケーションの中でこそ確かめることができる、と考えました。

今回のインターンシップは、参加者にとっては初めての苦労が多かったと思います。

しかし、初対面の人たちと大きいチームを組み、自分の技術を最大限発揮しなければ達成できない、というシチュエーションも、エンジニアの世界ではよくある光景です。

ちなみに、インフィニットループでは、20人程度のチームを組むこともありますし、お客様のチームも含めると、50人、100人近い仲間でプロジェクトを行うことは少なくありません。

## 「VR Arcade」

今回製作してもらったVRプラットフォームをご紹介します。

SteamVRからゲームを立ち上げ、プラットフォームにログインすると、VR空間に仮想のゲームセンターがあり、いくつかの体感ゲームをプレイすることができます。

実装にはUnity©︎を利用しました。

VRゲームは複数人でマルチプレイができます。

通信はLANで接続されたマシン同士で行います。

実装にはUnity NetworkingのLobbyを採用しました。

VRアバターには、VRM©︎フォーマットに準拠したモデルが使えます。

このモデルは、AWS©︎ S3で署名つきURLにより都度配信することで、簡易的なデータ保護を行う、という仕様になっています。

ゲームプレイの成績によって、UNYというゲーム内仮想通貨を稼ぐことができます。

このプラットフォームは、Web上にUNYの獲得量に基づくランキングを公開しています。

ウェブサイトはAWS上に構築されており、RDS + EC2 + docker(PHP,Laravel)で実装しています。

VR空間には視聴者がコメントを降らせることができます。

コメントのメッセージングの実装にはAWS©︎ SQSを使いました。

Unityクライアントでコメントキューを非同期ポーリングしています。

VRコントローラのメニューを開くと、左手に円形UIが出現します。

このUIからゲームの選択やプレイの停止が行えます。

VRでも操作しやすいようにするため、階層型になっています。

ビジュアル班は、主にMax©︎を使ってゲーム内アセットを作成しました。

3Dに関して全くの初心者も参加していましたが、経験のある参加者が一つだけ作り方をレクチャーしたあとは、各自がおもいおもいの3Dアセットを作っていました。

聞いたところ、最初に作るのはマイナスドライバーが良いそうです。

確かに、言われてみれば適度に複雑・・という造形をしています。

私は、マイナスドライバーに対する見方が変わりました。

コミュニケーションにはSlackを使いました。

データの受け渡しにはunitypackageでパッケージングしてLAN内のファイルサーバやSlackを経由して受け渡しました。

## 仕様の決め方

上記の仕様を決定するプロセスは参加者主導で行いました。

もともと「VRゲームを作る」というお題があったこともあり、技術的なアイデアは全て実装者があらかじめ持っていました。

その上で、メンターから利用できそうな技術を紹介したり、情報収集をサポートしたり、技術的なトレードオフを整理して、最終的に何を採用するかは参加者に都度決めてもらいました。

今回採用した技術に詳しい方であればお気づきかもしれませんが、短期間の開発を実現するために、様々なトレードオフがあり、結果として上記のような形に収まっています。

結果的に、テストプレイ中にマルチプレイのバグが見つかるが直しきれない、というところで時間切れになりました。

とはいえ、具体的なアイデアは全て実現できており、プロトタイプとしての品質は十分満たしていました。

（今回は、一部のソフトウェアライセンスの都合上、弊社で公開できなかったのが残念でしたので、来年度はこの点も考慮したいと思います。）

## 引き出す仕事

20世紀に労働の形は大きく変化しました。

仮に、それ以前の労働を「生きるための仕事」と呼びましょう。

飢えをなくしたり、社会衛生を改善して、みんなが長生きするためにする仕事です。

まだ世界に貧困はありますが、このまま文明が発展すれば、全ての人が仕事をしなくても暮らしていけるようになるかもしれません。

そして代わりに、アーティスト、芸能人、Youtuberといった仕事が増えていきます。

ところで「愉快な仕事」は、誰でも「愉快」に取り組める仕事でしょうか。

私はちょっと苦手です。

面白さをゼロから生み出すのは、とても難しいし、仕事ですから、極端に言えば「愉快になれ」と常に言われているような、強迫観念にとらわれそうになります。

「趣味を仕事にする」のは難しいものです。

もちろん、トップアーティストはそんなまどろっこしいことを考えずにどんどん面白さを生み出していきます。

ただし、一度走り始めると立ち止まることのできない危うさがあります。

時間とともに現れる「スランプ」や「流行」はあなたを待ってくれません。

「愉快な仕事」だけでは、疲れそうです。

もう少し誰にでもできる仕事はないでしょうか。

私はエンジニアが一つのあり方だと考えます。

正確には、エンジニアリングが「生きるための仕事」と「愉快な仕事」のあいだにある、と考えています。

ディズニーランドにはイマジニアと呼ばれる集団がおり、ありとあらゆる最先端の技術を総合して、アトラクションを作ります。

現代では、ハリウッドの手品師やM-1に出てくるお笑い芸人も、映像や音楽のイリュージョンを使うために、少なからずエンジニアリングをしています。

そもそも、我々の御先祖さまは、海や川、密林やサバンナで、いつもエンジニアリングをしていました。

「2001年宇宙の旅」には、サルが道具を使い始めるシーンがありますが、注目すべきは集団で狩りを行い、道具はその過程にすぎません。

エンジニアは、世界が突きつけてくる問題に対して、とりあえずの答えをその世界から引き出します。

状況はどんどん変わります。

私たちは偶然性に取り囲まれていて、細かい失敗を積み重ねながら前に進みます。

私はこの、ジャングルで暮らすような、ハンターのような生き方が気に入っています。

新しい技術を試すことは、怪しい色をした木の実を食べてみることと似ています。
